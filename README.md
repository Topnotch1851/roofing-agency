# Apex Roofing — Premium Demo Template

A production-grade, conversion-engineered roofing website built as a cold-outreach demo asset. Branded as **"Apex Roofing — Dallas, TX"** but architected to be rebranded for any roofing client in under 30 minutes.

Built with Next.js 15 (App Router) + Tailwind CSS v4 + Framer Motion + Phosphor icons. Engineered to meet every 2026 competitive standard: sub-2s mobile load, tap-to-call above the fold, full LocalBusiness schema, service-area SEO pages, FAQ schema, 3-field quote form with instant SMS hook.

---

## Quick start

```bash
npm install

npm run build # production build

```

### Design system

- **Vibe**: Editorial Luxury × Soft Structuralism. Warm cream paper, deep tinted ink, rust accent, amber CTA pop.
- **Colour**: All tokens in OKLCH. No `#000` or `#fff` anywhere. Tinted toward brand hue.
- **Typography**: `Sora` (display) + `Geist Sans` (body) + `Geist Mono` (labels). Inter is banned.
- **Motion**: Framer Motion with custom cubic-bezier curves (`ease-haptic`, `ease-out-quart`). Respects `prefers-reduced-motion`.
- **Cards**: Double-bezel architecture (outer shell + inner core with concentric radii) on key surfaces.
- **CTAs**: Button-in-button trailing icon pattern. Magnetic hover.
- **Layout**: `max-w-[1320px]`, generous `py-24`–`py-32` section padding. `min-h-[100dvh]` on full-height sections (no `h-screen` bug).

### SEO & schema

- **JSON-LD**: `RoofingContractor`, `LocalBusiness`, `Service`, `FAQPage`, `BreadcrumbList` — all wired to single source of truth in `lib/config.ts`.
- **Metadata API**: Per-page title, description, canonical, OG tags.
- **Sitemap & robots**: Auto-generated.
- **Service-area pages**: 800+ words of unique substantive copy each.
- **FAQ schema**: On home + every service page (improves AI/voice search citation rates).

### Conversion stack

- Sticky tap-to-call mobile bar (always visible, thumb-zone bottom).
- Phone number visible in desktop header without scrolling.
- 3-field quote form (Name, Phone, Address, service-needed pills).
- Instant SMS confirmation stub (Twilio integration point in `app/api/quote/route.ts`).
- Financing transparency ("From $89/month") prominent.
- Live Google review section with aggregate rating.
- Trust strip with 6 certifications above the fold.
- 25-year guarantee called out 11+ times across the site.

## Rebrand for a new client in 30 minutes

The entire site is driven by two files. Change them and the whole site updates.

**1.** Edit `lib/config.ts`:
```ts
export const site = {
  name: "Client Name Roofing",
  legalName: "Client Name Roofing LLC",
  phone: "(555) 123-4567",
  email: "estimates@clientname.com",
  address: { street: "...", city: "...", region: "TX", postal: "...", ... },
  license: "TX Reg #...",
  rating: { value: 4.9, count: 187 },
  // ...
};
```

**2.** Edit `lib/content.ts`:
- Update `services` (4 entries) — copy + FAQs + materials specific to client
- Update `areas` (3 entries) — replace cities with client's service areas
- Update `reviews` (6 entries) — paste real Google reviews
- Update `generalFaqs`

**3.** Optionally retune the palette in `app/globals.css`:
- `--rust` — primary accent (default warm rust orange)
- `--amber` — CTA colour (default warm amber)
- `--ink` — body/text base
- `--paper` — background

**4.** Replace `public/og.png` and `public/favicon.ico` with client branding.

**5.** Replace placeholder photo URLs (`https://picsum.photos/seed/...`) with real client photos. Search the codebase for `picsum.photos` to find each location. Recommended replacements:
- Hero photo (1200×960 portrait crop)
- 4 gallery photos (varied aspect ratios)
- 4 service detail hero photos (16:9)
- 3 area page hero photos (16:9)
- 1 about page team photo (16:9)
- 4 blog thumbnails

## Pre-launch checklist

- [ ] Update `site.url` in `lib/config.ts` to the live domain
- [ ] Wire `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`, `TWILIO_FROM` env vars (see `app/api/quote/route.ts`)
- [ ] Wire `RESEND_API_KEY` for internal lead notifications
- [ ] Replace `public/og.png` (1200×630) and favicon
- [ ] Replace placeholder photos with real project imagery
- [ ] Verify NAP (name, address, phone) matches Google Business Profile exactly
- [ ] Add Google Places API key + wire live reviews (currently using static high-quality placeholder reviews)
- [ ] Set up Google Search Console + submit `/sitemap.xml`
- [ ] Verify schema in [Rich Results Test](https://search.google.com/test/rich-results)
- [ ] Run Lighthouse audit (target 95+ Performance, 100 SEO, 95+ Accessibility)
- [ ] Confirm `.com` domain (not `.net` — see 2026 standard)
- [ ] Confirm SSL/HTTPS active (Vercel default)
